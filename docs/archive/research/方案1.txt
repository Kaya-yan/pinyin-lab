# PinyinLab 汉语动态舌位可视化教学平台
## 完整项目方案

---

## 一、项目定位与核心原则

**项目名称**：PinyinLab（汉语动态舌位可视化教学平台）

**核心定位**：面向海外低龄汉语学习者的纯前端发音辅助工具。以"标准音"对冲"网络梗音"，用可视化舌位视频替代平面口型示范。

**设计原则**：
- **极简**：无登录、无权限、无后端、打开即用
- **学术**：界面风格克制、专业，类似实验室仪器
- **极速**：静态导出，首屏加载 < 1 秒，印尼课堂弱网环境可用
- **可传播**：每个查询结果都有独立 URL，扫码即用

---

## 二、功能架构

平台仅包含**两个页面**，所有功能围绕"查汉字→看舌位→找片段"这一核心链路展开。

### 2.1 发音实验室页（`/lab`）

**核心链路**：搜索汉字 → 自动转拼音 → 顺序播放舌位视频 → 推荐关联配音片段

| 功能模块 | 具体描述 |
|---------|---------|
| **万能搜索** | 顶部搜索框输入任意汉字或词语（如"冰淇淋"），前端实时转拼音 |
| **拼音拆解条** | 将词语拆解为带声调拼音卡片（如 bīng / qí / lín），当前播放项高亮，点击可跳转 |
| **舌位视频播放器** | 根据拼音自动匹配本地视频，顺序播放；支持 0.5x 慢放、循环当前音节、进度条音节标记 |
| **连续语流模拟** | 多音节词语自动顺序播放，视频间 0.2 秒淡入淡出，进度条用小竖线标记音节切换点 |
| **关联片段推荐** | 根据当前词语的韵母/声母，自动筛选并展示片段库中匹配的 B 站配音卡片 |
| **URL 分享** | 支持 `/lab?word=冰淇淋`，打开直接定位到对应结果 |

### 2.2 片段库页（`/clips`）

**核心链路**：按拼音筛选 → 查看片段元数据 → 预览配音片段

| 功能模块 | 具体描述 |
|---------|---------|
| **拼音字母网格** | 声母/韵母按钮矩阵，点击筛选含该拼音的所有片段 |
| **片段卡片列表** | 展示封面、标题、时长、起播时间、标签 |
| **B站嵌入预览** | 点击卡片弹出模态框，内嵌 B 站 iframe 播放，叠加教学聚焦卡片 |

### 2.3 明确不做（已做减法）

- ❌ 用户登录/注册系统
- ❌ 教师端/学生端区分
- ❌ 录音跟读与波形对比
- ❌ 学习数据看板
- ❌ 发音偏误数据库
- ❌ AI 语音评测打分
- ❌ 后台管理系统

## 三、前后端技术架构

### 3.1 架构总览

采用 **纯静态前端架构（SSG）**，构建时生成纯 HTML/CSS/JS，部署至 CDN。

```
用户浏览器
    │
    ▼
Next.js 14 静态导出站点 (SSG)
    │
    ├── 页面层: /lab, /clips
    │
    ├── 逻辑层: pinyin-pro (汉字转拼音)
    │           视频顺序控制器
    │           URL 参数解析器
    │
    ├── 静态数据层:
    │       /data/pinyinMap.json  → 拼音→视频映射
    │       /data/clips.json      → 片段元数据
    │       /public/videos/*.mp4  → 舌位视频资产
    │       /public/covers/*.jpg  → 片段封面图
    │
    └── (播放时) → B站 iframe CDN (player.bilibili.com)
```

### 3.2 技术选型

| 模块 | 技术方案 | 选型理由 |
|------|---------|---------|
| 前端框架 | Next.js 14 (App Router) | 静态导出生成纯静态文件；文件系统路由支持 `/lab?word=xxx` |
| 开发语言 | TypeScript | 类型安全，维护 JSON 数据结构时减少错误 |
| 拼音引擎 | pinyin-pro | 体积 400KB，浏览器内实时转拼音，零后端依赖，支持多音字 |
| 视频播放 | HTML5 原生 `<video>` | 无需第三方库，支持倍速/循环/预加载，API 极简 |
| B站嵌入 | iframe + 官方播放器 | 官方稳定 CDN，支持起播时间/关弹幕/自动播放参数 |
| 样式方案 | Tailwind CSS | 原子化 CSS，快速实现响应式与学术风设计 |
| 部署平台 | Vercel | 全球 CDN、HTTPS 默认、自定义域名、中国大陆可访问 |

## 五、UI/UX 设计规范

### 5.1 设计原则

- **简洁感**：大留白、零阴影、细线分割、无渐变
- **学术风**：衬线体标题、克制配色、网格对齐、印刷质感
- **可用性**：大按钮（最小 48px 高）、大拼音（56px）、教室投影清晰可读

### 5.2 色彩规范

| 名称 | 色值 | 用途 |
|------|------|------|
| 主色 Primary | `#2C5282` | 当前播放边框、主按钮、进度条、链接、导航激活态 |
| 辅助色 Secondary | `#4A5568` | 次要按钮、标签文字、图标 |
| 背景色 Background | `#FAFAF8` | 页面全局底色（象牙白） |
| 卡片色 Surface | `#FFFFFF` | 内容卡片、弹窗底色 |
| 文字主色 Text | `#1A202C` | 标题、拼音、正文 |
| 文字次色 Text Muted | `#718096` | 说明文字、未激活拼音、元信息 |
| 强调色 Accent | `#B7791F` | **仅用于**教学聚焦卡片左侧边框 |
| 边框色 Border | `#E2E8F0` | 卡片边框、分割线、表格线 |
| 当前高亮 Highlight | `#EBF4FF` | 当前播放拼音卡片背景 |

### 5.3 字体规范

| 层级 | 字号 | 字重 | 字体 | 用途 |
|------|------|------|------|------|
| 页面标题 | 24px | 600 | Noto Serif SC | 顶部 Logo 旁平台名 |
| 拼音展示 | 56px | 700 | Noto Sans SC | 当前播放拼音（教室后排可见） |
| 汉字展示 | 28px | 500 | Noto Sans SC | 搜索结果汉字 |
| 按钮文字 | 16px | 500 | Noto Sans SC | 所有可点击按钮 |
| 正文 | 15px | 400 | Noto Sans SC | 说明文字、片段描述 |
| 标签/元信息 | 13px | 500 | Noto Sans SC | 拼音类型标签、时长 |

**字体栈**：
```css
font-family: 'Noto Serif SC', 'Noto Sans SC', 'PingFang SC', 'Microsoft YaHei', sans-serif;
```

### 5.4 间距与形状

| 属性 | 规范 |
|------|------|
| 圆角 | 4px（按钮）、8px（卡片/视频）、12px（弹窗） |
| 卡片内边距 | 24px |
| 模块间距 | 32px-48px |
| 边框 | 1px solid `#E2E8F0`，零阴影 |
| 最大内容宽 | 960px，居中布局 |

**组件规格**：

**① 顶部导航**
- 高度 56px，背景 `#FFFFFF`，底部边框 `1px solid #E2E8F0`
- 左侧：平台名 "PinyinLab"，字体 Noto Serif SC，24px，`#1A202C`
- 右侧：搜索框（宽 280px，高 40px，圆角 4px，边框 `#E2E8F0`，聚焦时边框变 `#2C5282`）+ "片段库"文字链接（`#2C5282`，hover 下划线）

**② 拼音拆解条（PinyinStrip）**
- 整体居中，上方显示汉字（28px，`#1A202C`），下方显示拼音（56px，`#1A202C`）
- 每个音节为一个卡片：
  - 尺寸：auto × 80px，内边距 `16px 32px`，圆角 8px
  - **当前播放**：背景 `#EBF4FF`，边框 `2px solid #2C5282`，文字 `#2C5282`
  - **待播放**：背景 `#FFFFFF`，边框 `1px solid #E2E8F0`，文字 `#718096`
  - 卡片间距：16px
- 交互：hover 时边框变 `#2C5282`（过渡 150ms），点击后视频切换

**③ 视频播放器（VideoPlayer）**
- 容器：最大宽度 800px，居中，圆角 8px，边框 `1px solid #E2E8F0`
- 视频：HTML5 `<video>`，宽高比 16:9
- 控制栏（自定义，视频下方）：
  - 背景 `#FFFFFF`，上边框 `1px solid #E2E8F0`，高度 56px
  - 按钮：40×40px，图标 `#4A5568`，hover `#2C5282`
  - 进度条：高 4px，已播放 `#2C5282`，未播放 `#E2E8F0`，圆角 2px
  - 音节标记：进度条上方小竖线（`#B7791F`，宽 2px，高 8px）

**④ 片段推荐卡片（ClipCard）**
- 网格：3 列，间距 24px
- 卡片：背景 `#FFFFFF`，圆角 8px，边框 `1px solid #E2E8F0`
- 封面图：宽高比 16:9，圆角 8px 8px 0 0
- 信息区：padding 16px，标题 16px `#1A202C`，时长 13px `#718096`
- 按钮：全宽，高 40px，背景 `#FFFFFF`，边框 `1px solid #2C5282`，文字 `#2C5282`，圆角 4px
- hover：按钮背景变 `#2C5282`，文字变白（过渡 150ms）

## 七、部署与性能优化

| 优化项 | 具体措施 |
|-------|---------|
| 静态导出 | `next.config.js` 设置 `output: 'export'`，构建产物为纯静态文件 |
| 视频压缩 | 舌位视频使用 H.264 编码，分辨率 720p，单文件控制在 < 2MB |
| 预加载策略 | 当前播放视频 `preload="auto"`，下一个视频 `preload="metadata"` |
| 图片优化 | 片段封面使用 WebP 格式，Next.js Image 组件自动优化 |
| CDN 缓存 | Vercel 默认缓存静态资源 1 年，JSON 文件设置 `Cache-Control: public, max-age=3600` |
| 懒加载 | 片段库封面图使用 `loading="lazy"`，首屏不加载不可见内容 |

## 八、给 Claude Code 的完整开发提示词

> 请基于以下要求开发一个汉语拼音舌位可视化教学平台，项目名 PinyinLab。整体风格为**简洁学术风**，追求类似学术工具/实验室仪器的克制与专业感，拒绝活泼卡通感。
>
> **技术栈**：Next.js 14 (App Router) + TypeScript + Tailwind CSS + pinyin-pro
>
> **设计系统（严格遵循）**：
> - 全局背景色 `#FAFAF8`（象牙白），卡片背景 `#FFFFFF`，卡片边框 `1px solid #E2E8F0`，**零阴影**
> - 主色 `#2C5282`（学术蓝），用于当前播放状态、主按钮、进度条、链接
> - 强调色 `#B7791F`（赭石），**仅用于**教学聚焦卡片的左侧边框，不得大面积使用
> - 文字主色 `#1A202C`，文字次色 `#718096`
> - 圆角规范：小 4px（按钮）、中 8px（卡片/视频）、大 12px（弹窗）
> - 字体：标题用 `font-serif`（Noto Serif SC），正文和拼音用 `font-sans`（Noto Sans SC）
> - 拼音展示字号 56px、字重 700，确保教室投影清晰可读
> - 按钮最小高度 48px，保证儿童可点击
> - 最大内容宽度 960px，居中布局
>
> **页面与功能**：
> 1. 创建两个页面：`/lab`（发音实验室）和 `/clips`（片段库）
> 2. `/lab`页：顶部导航栏（高56px，白底，底边框1px #E2E8F0，左侧衬线体Logo"PinyinLab"，右侧搜索框+片段库链接）。搜索框输入汉字后，使用 pinyin-pro 实时转拼音并带声调显示在搜索框下方（汉字28px，拼音56px）。下方为拼音拆解条：每个音节为一个卡片（当前播放卡片背景 `#EBF4FF`、边框 `2px solid #2C5282`，待播放卡片白底灰边框），卡片可点击切换。再下方为视频播放器（最大宽800px，16:9，圆角8px，细线边框，无阴影），使用 HTML5 video 标签顺序播放对应视频文件（来自 `/public/videos/` 目录），支持播放/暂停、0.5x 慢放、循环当前音节，视频切换时有 0.2 秒淡入淡出。播放器下方进度条用小竖线（`#B7791F`）标记音节切换点。视频区下方展示关联 B 站片段卡片网格（3列，间距24px），卡片白底细线边框圆角8px，封面图+标题+时长+预览按钮，hover 时预览按钮蓝底白字。
> 3. 点击片段卡片的"预览"按钮弹出模态框：遮罩 `rgba(26,32,44,0.5)`，内容区最大宽880px 白底圆角8px 细线边框。顶部标题栏带关闭按钮。内嵌 B 站 iframe（参数：`danmaku=0` 关闭弹幕、`t=指定时间` 起播、`autoplay=true`、`high_quality=1`）。iframe 下方叠加"教学聚焦卡片"：背景 `#FAFAF8`，左侧边框 `3px solid #B7791F`，圆角4px，内显示练习重点文字，底部有 B 站原链接。
> 4. `/clips`页：顶部同样导航栏。下方为拼音字母网格按钮（48×48px，圆角4px，白底灰边框灰文字），点击后激活态为蓝底白字。下方展示筛选后的片段卡片网格，样式与 lab 页一致。
> 5. 支持 URL 参数分享：`/lab?word=冰淇淋` 打开直接显示对应结果。
>
> **数据文件**：
> - `/data/pinyinMap.json`：拼音到视频文件映射（含 type/video/duration/description）
> - `/data/clips.json`：B站片段元数据（id/title/cover/bvid/startTime/duration/tags/focus/source/uploader）
> - `/public/videos/`：存放舌位视频
> - `/public/covers/`：存放片段封面图
>
> **约束**：无登录系统、无后端API、无数据库、无阴影、无渐变背景、纯静态导出（`output: 'export'`）、追求首屏极速加载。

## 九、案例文本撰写建议（比赛文档用）

若需在案例文本中描述该平台，建议采用以下学术化表述框架：

> **数智赋能：AI 动态舌位可视化平台的构建**
>
> 针对传统语音教学中"平面口型示范无法呈现口腔内部运动"的痛点，本案例依托生成式人工智能技术，构建了一款面向海外低龄学习者的纯前端发音辅助平台。该平台以 HTML5 视频序列播放器为核心，将离散的音位舌位视频（如 ai、ao、ing 等）通过拼音自动映射与顺序串联技术，实现多音节词语的"伪连续语流"可视化呈现——输入"冰淇淋"后，系统自动拆解为 bīng-qí-lín 并顺序播放对应舌位视频，配合 0.5x 慢放与音节跳转功能，使学生能够逐帧观察协同发音过程中的舌位动态变化。
>
> 在内容层面，平台整合了经教师筛选的中文正向文化配音片段（来源于 Bilibili 等公开平台），通过 iframe 嵌入式播放与教学蒙层设计，将网络视频资源转化为结构化教学素材。教师可根据当日所学音位，一键调取含目标拼音的动画片段，引导学生进行沉浸式配音练习，以"标准音输入"对冲"网络梗音"的负迁移干扰。
>
> 技术层面，平台采用 Next.js 静态导出架构，所有数据以 JSON 静态文件形式存在，零后端依赖，确保印尼课堂弱网环境下的极速加载与离线可用性。拼音转换引擎基于 pinyin-pro 实现浏览器端实时汉字转拼音，无需服务器交互，兼顾了数据隐私与响应速度。
