# PinyinLab 项目全盘审查报告

## 一、项目概述

**项目名称**：PinyinLab（汉语动态舌位可视化教学平台）  
**项目定位**：面向海外低龄汉语学习者的纯前端发音辅助工具  
**核心目标**：以"标准音"对冲"网络梗音"，用可视化舌位视频替代平面口型示范  
**参赛背景**：2026年山东大学第二届研究生国际中文教育案例大赛（数智赋能赛道）

---

## 二、项目架构审查

### 2.1 技术栈评估

| 模块 | 技术方案 | 评估 |
|------|---------|------|
| 前端框架 | Next.js 14 (App Router) | ✅ 静态导出支持完善，SSG 首屏加载快 |
| 开发语言 | TypeScript | ✅ 类型安全，代码可维护性高 |
| 拼音引擎 | pinyin-pro v3.28.1 | ✅ 400KB 体积适中，支持多音字和声调 |
| 样式方案 | Tailwind CSS v3.4.1 | ✅ 原子化 CSS，开发效率高 |
| 视频播放 | HTML5 原生 `<video>` | ✅ 无需第三方库，API 极简 |
| B站嵌入 | iframe + 官方播放器 | ✅ 官方 CDN 稳定，参数可控 |
| 部署方案 | Vercel (静态导出) | ✅ 全球 CDN，中国大陆可访问 |

### 2.2 项目结构

```
pinyin-lab/
├── app/                    # Next.js App Router
│   ├── lab/page.tsx        # 发音实验室页（核心页面）
│   ├── clips/page.tsx      # 片段库页
│   ├── layout.tsx          # 根布局（SEO、无障碍）
│   └── globals.css         # 全局样式 + Tailwind
├── components/             # 可复用组件
│   ├── Navbar.tsx          # 导航栏
│   ├── PinyinStrip.tsx     # 拼音拆解条
│   ├── VideoPlayer.tsx     # 视频播放器（核心组件）
│   ├── ClipCard.tsx        # 片段卡片
│   └── BilibiliModal.tsx   # B站弹窗
├── lib/                    # 工具函数
│   ├── pinyin.ts           # 拼音解析逻辑
│   └── types.ts            # TypeScript 类型定义
├── data/                   # 静态数据
│   ├── pinyinMap.json      # 拼音→视频映射
│   └── clips.json          # 片段元数据
└── public/                 # 静态资源（视频/GIF/封面）
```

**评估**：项目结构清晰，职责分离合理，符合 Next.js 最佳实践。

## 四、代码质量审查

### 4.1 TypeScript 类型安全

- ✅ 定义了 `PinyinSegment`、`SubSegment`、`Clip` 等接口
- ✅ 使用了 `type` 导入
- ✅ `tsconfig.json` 启用了 `strict: true`
- ⚠️ **clips.json 导入类型**：`clipsData.clips` 使用了 `as Clip[]` 类型断言，建议完善 JSON 类型定义

### 4.2 拼音解析逻辑 (`lib/pinyin.ts`)

**优点**：
- ✅ 使用 pinyin-pro 进行汉字转拼音
- ✅ 支持声调符号（toneType: "symbol"）
- ✅ 声母提取逻辑完善（处理了 zh/ch/sh 等双字母声母）
- ✅ 韵母归一化（处理 j/q/x/y 后的 ü 恢复、y/w 前缀处理）
- ✅ 别名规范化（iou→iu、uei→ui、uen→un）

**待优化项**：
- ⚠️ **零声母处理**：y、w 作为零声母标记，当前代码中 `initialsList` 包含 y/w，但 pinyinMap.json 中无对应数据，可能导致解析异常
- ⚠️ **错误处理**：`parseWord` 函数对无拼音结果的字符（如标点、数字）处理不够健壮

### 4.3 视频播放器逻辑 (`VideoPlayer.tsx`)

**优点**：
- ✅ 使用 `useMemo` 缓存 flatSubs，避免重复计算
- ✅ 视频切换时同步更新 playbackRate 和 loop 状态
- ✅ 自动播放下一个 sub-segment
- ✅ 同步 activeIndex 和 flatIdx

**待优化项**：
- ⚠️ **视频预加载策略**：当前使用 `preload="auto"`，对于大体积 GIF/MOV 资源可能造成性能问题
- ⚠️ **资源缺失处理**：当 video/gif 均为 null 时显示"暂无视频资源"，但无更友好的引导
- ⚠️ **循环逻辑**：`isLoop` 状态下视频 ended 事件处理可能导致播放状态不一致

### 4.4 无障碍 (A11y) 审查

**已实现**：
- ✅ 跳转链接（skip link）
- ✅ ARIA 标签（role、aria-label、aria-selected、aria-pressed）
- ✅ 焦点可见样式（focus-visible）
- ✅ 键盘导航支持（Escape、Tab、Arrow keys）
- ✅ 减少动画偏好支持（prefers-reduced-motion）

**待完善**：
- ⚠️ **视频播放器无障碍**：进度条 role="slider" 缺少 aria-valuetext，屏幕阅读器用户无法获知当前播放位置
- ⚠️ **拼音卡片**：当前使用 `<button>` 包裹，但内部有 56px 大文字，可能在小屏幕上溢出

## 七、问题汇总与改进建议

### 🔴 严重问题（需立即修复）

1. **GIF 教学聚焦未实现**
   - 影响：核心教学功能缺失，无法实现"口腔侧剖面动画"展示
   - 建议：在 VideoPlayer 组件中添加左侧 64px GIF 展示区，声母播放时显示对应 GIF

2. **封面图缺失**
   - 影响：片段卡片视觉吸引力不足，用户体验差
   - 建议：为每个片段制作/获取封面图，或使用 B站视频封面 API

### 🟡 中等问题（建议修复）

3. **色彩偏差**
   - 实际 `accent` 色 `#8A5C17` 与方案 `#B7791F` 不一致
   - 实际 `text-muted` 色 `#5A6B80` 与方案 `#718096` 不一致
   - 建议：统一调整为方案定义色值

4. **零声母处理**
   - y、w 作为零声母标记，当前解析逻辑可能产生异常
   - 建议：在 `extractInitial` 中特殊处理零声母情况

5. **视频预加载策略**
   - 当前 `preload="auto"` 对大体积 GIF/MOV 不友好
   - 建议：根据资源类型设置不同预加载策略（MP4 用 auto，MOV/GIF 用 metadata）

### 🟢 优化建议（可选）

6. **添加 Service Worker 缓存**
   - 实现离线可用，提升弱网环境体验

7. **添加加载状态提示**
   - 视频/GIF 加载时显示 loading 状态

8. **片段卡片封面占位**
   - 在获取真实封面图前，使用拼音标签或颜色块作为占位

9. **SEO 优化**
   - 为每个拼音页面生成静态路径（如 `/lab/bing`）

## 八、总体评价

### 8.1 优势

1. **架构简洁**：纯静态前端，无后端依赖，部署简单
2. **代码质量高**：TypeScript + 严格模式，类型安全
3. **设计规范**：遵循学术风设计系统，界面克制专业
4. **功能完整**：核心链路（搜索→转拼音→播放视频→推荐片段）已打通
5. **无障碍支持**：ARIA 标签、键盘导航、焦点管理完善

4. **边缘情况处理**：零声母、资源缺失等场景处理不够健壮
